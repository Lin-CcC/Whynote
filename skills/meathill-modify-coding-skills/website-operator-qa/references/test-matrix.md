# 运营视角网站人工测试矩阵

按站点类型挑选适用项，不要机械全量执行。目标是验证“这个站点能不能真的拿去对外服务”，不是覆盖所有技术实现。

## 1. 测试准备

- 明确前台站点、CMS/后台地址、测试账号、支持语言、是否允许创建临时数据
- 明确站点类型：品牌站、内容站、B2B 产品站、服务站、活动站、电商站
- 明确主转化目标：询价、联系、订阅、预约、下单、下载资料、提交工单
- 明确清理策略：测试数据是否必须删除，是否允许发真实邮件或 webhook

## 2. 前台站点

### 首页与导航

- 首屏是否能快速说明网站提供什么服务
- 一级导航是否可理解，是否有重复、死链、层级混乱
- 顶栏/页脚联系方式是否完整且一致
- 首页 CTA 是否指向真实可完成的路径

### 内容发现

- 站内搜索是否能搜到明明存在的内容
- 分类页/列表页是否能支持浏览和筛选
- 卡片的标题、摘要、图片、发布日期、标签是否完整
- 面包屑、相关推荐、返回路径是否合理

### 详情页与信任感

- 产品/文章/服务详情页是否存在空白模块、空表格、占位文案
- 图片、规格、FAQ、下载资料、法律文案是否可信
- slug、标题、发布日期、语言版本是否正确
- 是否存在明显损伤专业感的问题，例如 `undefined`、404 占位图、重复菜单

### 转化路径

- 从首页、列表页、详情页是否都能找到转化入口
- 询价/联系/订阅/预约表单是否可填写、可提交、错误提示可理解
- 提交后是否有成功反馈、失败反馈和后续预期说明
- 网络请求是否发到正确环境，而不是 `localhost` 或测试域名

### 法务与辅助页面

- About、Contact、Privacy、Terms、FAQ、Support 是否可对外使用
- 是否仍保留 `coming soon`、`being prepared` 或模板站占位文案
- 联系我们页面是否真的承接咨询，而不是只有标题

### 多语言

- 语言切换入口是否明显
- 切换后是否只是换路由，还是内容真的翻译了
- 未翻译内容的回退策略是否合理
- 不同语言下的表单、日期、CTA、SEO 文案是否一致

### 移动端

- 导航是否正确折叠
- 关键 CTA 是否仍可见
- 表单、弹层、图片轮播、列表卡片是否可操作
- 是否出现横向滚动、遮挡或点击区域过小

## 3. CMS / 后台

### 信息架构

- 登录后能否快速找到内容、业务、设置、消息等入口
- 集合命名、字段分组、按钮文案是否符合运营认知

### 列表页

- 列表是否能靠标题/名称识别记录
- 是否有大量 `<No Title>`、`Untitled`、纯 ID 记录
- 搜索、筛选、排序、分页是否帮助运营定位内容
- 空状态是否能指导下一步操作

### 新建与编辑

- 最小必填集是否清晰
- 校验提示是否明确指出缺了什么
- 保存后是否有明确反馈
- 再次进入编辑页时数据是否完整回显

### 状态流转

- 草稿、待审核、已发布、归档等状态是否易懂
- 状态切换后是否确实保存
- 后台是否允许明显不完整的数据被发布

### 媒体与资源

- 上传、选择、复用媒体是否顺畅
- Alt、标题、描述等字段是否合理要求
- 删除媒体后是否有引用风险提示

### 页面配置与站点设置

- About、Contact、Homepage、Menu、Footer、Site Settings 是否能被运营定位
- 后台填写的内容是否会稳定反映到前台
- 同一份设置是否在不同页面渲染一致

### 预览与多语言编辑

- Live Preview 是否能打开正确地址
- 预览是否展示当前草稿而不是旧发布内容
- 切换未翻译 locale 时，标题和上下文是否仍可识别
- AI 翻译或手工翻译入口是否可理解

### 业务后台

- 询盘、联系消息、订阅者、工单、订单等后台是否支持日常处理
- 字段是否按业务类型收敛，而不是把无关字段全部展示出来
- 状态更新、删除、归档、恢复是否顺畅

## 4. 跨链路检查

- 前台用户行为是否沉淀到后台，例如询盘、联系消息、搜索日志、订阅者
- CMS 已配置内容在前台是否真的被渲染
- 数据质量问题是否来自内容本身，还是 UI 允许发布不完整记录
- 控制台错误、资源 404、preview 404 是否对应可见业务问题

## 5. 测试后清理

- 删除本轮创建的草稿、测试图片、临时询盘、临时文章、临时产品
- 如果不能删除，记录名称、ID、路径和原因
- 报告中明确说明哪些问题是系统缺陷，哪些是现有内容脏数据
