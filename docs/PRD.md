# Whynote PRD

## 1. 产品概述

Whynote 是一个围绕提问式学习构建的节点化学习系统。它把学习过程中的问题、回答、总结、判断、资料引用和结构调整统一到同一棵节点树中，让用户在学习的同时自然生成可整理、可导出的笔记。

Whynote 首版不是通用思维导图工具，也不是普通聊天式 AI 笔记软件。它的核心目标是把“思考过程”变成“可持续编辑的结构化文档”。

## 2. 目标与边界

### 2.1 首版目标

- 验证“边学习边长笔记”的核心体验
- 验证统一节点模型是否足以承载学习计划、问答、总结、判断和资料引用
- 验证文本主视图与结构视图联动是否比传统聊天工具更适合长期学习
- 验证自动结构化结果是否能被用户轻量修正，而不是打断学习节奏

### 2.2 首版非目标

- 不做通用自由画布思维导图
- 不做公开商用级账号与平台代理层
- 不做云同步与多人协作
- 不做自动联网资料检索
- 不做 PDF / DOC 导出

## 3. 目标用户

首版优先服务个人自用场景，尤其是：

- 习惯通过提问来推进理解的人
- 需要在学习过程中不断重排笔记结构的人
- 既想要 AI 引导，又不想失去对自己笔记结构掌控权的人

## 4. 产品原则

- 节点统一：所有内容都以节点组织
- 类型清晰：节点统一，但语义必须明确
- 文本优先：文本区是主舞台，结构视图是辅助与重排工具
- 自动优先但可回退：系统先自动组织，用户再轻量修正
- 少 UI：只保留对学习链路真正必要的功能
- 过程与成稿分层：学习过程可见，但导出结果默认更像整理后的笔记

## 5. 核心产品模型

### 5.1 节点层级

Whynote 首版使用如下固定树结构：

```text
theme-root
├─ module
│  ├─ plan-step
│  │  ├─ question
│  │  │  ├─ question
│  │  │  ├─ answer
│  │  │  ├─ summary
│  │  │  └─ judgment
│  │  ├─ answer
│  │  ├─ summary
│  │  └─ judgment
│  └─ summary
└─ resource
   └─ resource-fragment
```

### 5.2 节点类型定义

- `theme-root`
  整个学习主题。根节点下直接子级只允许 `module` 和 `resource`。
- `module`
  一个学习方向模块。它既是结构单元，也是可单独导出的模块文档。
- `plan-step`
  学习计划步骤。模块内学习内容的主要容器。
- `question`
  问题节点。可作为主问题，也可作为追问或自动拆分出的子问题。
- `answer`
  回答节点。允许同一问题下存在多个回答节点。
- `summary`
  总结节点。允许阶段性总结与模块级总结共存。
- `judgment`
  判断 / 评估节点。用于判断用户回答或总结的准确性与完整性。
- `resource`
  资料源节点。位于全局资料区。
- `resource-fragment`
  资料切片或摘录节点，仅在需要稳定引用时按需生成。

### 5.3 节点标签

首版所有节点都允许挂标签。内建标签至少包括：

- `重要`
- `未理解`
- `待验证`
- `待整理`

模型层不限制标签只能作用于某类节点；显示层可对不同节点类型做轻重区分。

## 6. 模块与学习计划生成

### 6.1 入口

首版支持三种学习入口：

- 只输入主题
- 导入资料
- 导入资料并附带用户问题 / 学习偏好

第三种是推荐路径，因为它最容易生成针对性的学习结构。

### 6.2 学习模式

首版支持三种模式：

- `快速`
  - 2-3 个 `module`
  - 每个模块 2-3 个 `plan-step`
- `标准`
  - 3-5 个 `module`
  - 每个模块 3-5 个 `plan-step`
- `深度`
  - 4-7 个 `module`
  - 每个模块 4-6 个 `plan-step`

默认模式为 `标准`。

### 6.3 计划节点规则

- 学习计划进入节点树，`plan-step` 是正式节点
- `plan-step` 编辑时可见
- 导出默认隐藏 `plan-step` 标题，但保留其子内容顺序并并入模块正文
- 用户可显式选择包含完整学习路径

### 6.4 步骤完成状态

- `plan-step` 至少支持 `todo / doing / done`
- 完成状态以用户手动确认为准
- 系统可根据当前步骤下的问题、总结与判断情况给出“建议完成”提示
- 系统不自动将步骤置为完成

## 7. 问题与自动结构化

### 7.1 复合问题自动拆分

用户输入包含多个独立问题的复合问题时：

- 系统默认自动拆分
- 原句保留为父 `question`
- 子问题显式生成并可直接编辑
- 用户可拖拽、升降级或撤销拆分结果

### 7.2 子问题排序

- 优先按理解依赖排序
- 若系统对依赖关系低置信度，则退回原句顺序
- 用户手动排序优先于系统结果

### 7.3 追问规则

- 用户在学习中提出追问时，默认挂到当前 `plan-step` 或当前 `question` 下
- 子问题完成后，父问题仍保留，可继续回答、总结或判断

### 7.4 回答 / 总结 / 判断节点

- `answer / summary / judgment` 都是独立节点，不是问题内部的文本区块
- 同一问题下允许存在多个同类型节点，以保留迭代学习过程

## 8. 结构编辑

### 8.1 通用操作

首版树编辑使用统一的结构操作：

- `提升一级`
- `降低一级`
- 拖拽重排

### 8.2 根层模块语义

- 节点提升到根节点直接子级时，自动获得 `module` 语义
- 标题默认沿用当前节点标题
- 原有子树、标签、引用与顺序保留
- `resource` 系节点不参与此转换

### 8.3 产品意图

结构编辑是 Whynote 的核心能力之一，用于让用户以最少文本搬运成本重新组织自己的学习材料。

## 9. 资料区与引用

### 9.1 全局资料区

- 所有资料源统一放在根节点下的全局资料区
- 资料不直接复制到模块内部
- 模块、步骤、问题、总结通过引用连接资料

### 9.2 资料切片

`resource-fragment` 只在以下场景按需生成：

- 用户主动摘录
- 系统生成引用定位
- 搜索命中后需要稳定的片段引用

首版不做导入即全文切片。

### 9.3 引用关系

- 模块、计划步骤、问题、总结、判断都可以引用 `resource` 或 `resource-fragment`
- 引用关系用于对照学习、回跳定位与后续更严格的资料绑定评估

## 10. 搜索、定位与标签

### 10.1 搜索作用域

- 默认搜索范围是当前模块
- 搜不到时提示“当前模块无结果”
- 系统不自动扩大到全主题
- 用户可主动切换到“全主题”或“资料区”搜索

### 10.2 定位表现

- 当有多个命中结果时，使用类似编辑器定位栏的可视化相对位置提示
- 点击定位栏中的命中位置，可跳转到对应节点

### 10.3 标签检索

- 标签可作为搜索与筛选维度
- 导出按标签筛选时，应保留命中节点和必要祖先路径，避免上下文丢失

## 11. 导出规则

首版支持导出：

- 整个主题
- 单个模块
- 按标签筛选后的结果

格式仅限：

- Markdown
- TXT

默认导出行为：

- 隐藏 `plan-step` 标题
- 保留 `plan-step` 下的子内容顺序
- 生成更接近整理后笔记的文本

用户可显式选择导出完整学习路径。

## 12. 界面基线

### 12.1 文本主视图

- 文本区是学习主舞台
- 用户的大部分输入、修订、总结、判断发生在文本区

### 12.2 结构视图

- 用于展示节点结构
- 用于拖拽重排、升降级、切换关注范围
- 不是自由画布式思维导图

### 12.3 全局资料区

- 独立于模块结构存在
- 作为引用源与对照学习的资料容器

### 12.4 搜索与定位栏

- 搜索默认作用于当前模块
- 多命中结果通过定位栏显示相对位置与跳转入口

## 13. AI 设置基线

首版采用用户自填 OpenAI-compatible 配置，至少支持：

- `base URL`
- `API key`
- `model`

首版定位是个人模式，不把该方案定义为公开商用默认形态。

## 14. 存储基线

- `IndexedDB`
  - 工作区
  - 模块
  - 节点树
  - 标签
  - 引用关系
  - 资料元数据
- `localStorage`
  - 设置项
  - 最近打开状态
  - 界面偏好

## 15. 必须覆盖的场景

- 用户输入主题后，系统生成模块与计划步骤
- 用户导入资料并提出问题后，系统生成更有针对性的学习结构
- 用户输入复合问题后，系统自动拆分并继续逐个引导
- 用户通过提升 / 降低和拖拽重排结构
- 节点提升到根层后自动转为模块
- 用户导入资料后在模块中引用资料并生成按需切片
- 用户在当前模块内搜索、切全主题搜索、切资料区搜索
- 用户导出模块时默认隐藏步骤标题，但保留学习内容

## 16. 后续版本范围

以下能力明确进入后续版本，而不是首版目标：

- 自动资料检索
- PDF / DOC 导出
- 云同步
- 多设备共享
- 公开产品化的后端代理层
- 多人协作
- 更完整的资料对照阅读区
