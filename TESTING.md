# TESTING

本文档定义 Whynote 首版的测试基线。当前仓库还没有业务代码，因此本文档先约束测试范围、测试层次、推荐命令与验收重点，待工程初始化后落地具体实现。

## 目标

- 验证节点模型与结构编辑规则是否稳定
- 验证学习主链路是否符合产品定义
- 验证资料引用、搜索、导出与标签等辅助能力不会破坏主结构
- 验证本地存储与恢复逻辑是否可靠

## 测试栈基线

- 单元 / 集成测试：Vitest
- React 组件测试：Vitest + DOM 测试环境
- 浏览器级 smoke / E2E：Playwright（Chromium）
- 类型检查：TypeScript
- 构建验证：Vite build

首版浏览器测试只做轻断言 smoke，不提前锁死仍在演进中的交互细节。

## 推荐测试层次

### 1. 领域模型与纯逻辑

优先覆盖率最高，应接近 100%：

- 节点类型与层级校验
- `theme-root -> module / resource` 结构约束
- `module -> plan-step -> learning nodes` 的默认插入规则
- `提升一级 / 降低一级` 的树变换逻辑
- 根层提升后自动转 `module` 的语义转换
- 复合问题自动拆分与顺序计算
- 标签挂载与筛选逻辑
- 导出转换逻辑

### 2. 存储与恢复

- IndexedDB 中的结构化数据读写
- localStorage 中的设置项与最近状态保存
- 本地恢复后树结构、引用关系、标签、步骤状态保持一致

### 3. AI 编排与规则测试

使用 mock provider 或 contract test，避免测试真实大模型输出稳定性：

- 三种学习模式生成的模块 / 步骤粒度是否符合约束
- 复合问题自动拆分是否保留父问题
- 子问题排序在高置信度和低置信度两种分支下是否正确
- 系统只建议完成 `plan-step`，不自动改写为完成状态

### 4. 组件与交互测试

- 文本主视图与结构视图的同步
- 搜索命中后定位栏显示与跳转
- 标签显隐与节点高亮
- 计划步骤在编辑区可见、导出默认隐藏
- 资料引用回跳与摘录展示

## 首版必须覆盖的场景

### 学习结构生成

- 用户输入主题后，系统按 `快速 / 标准 / 深度` 之一生成模块，默认 `标准`
- `快速`：2-3 个模块，每个 2-3 个步骤
- `标准`：3-5 个模块，每个 3-5 个步骤
- `深度`：4-7 个模块，每个 4-6 个步骤

### 模块内学习

- 模块下先生成 `plan-step`
- 问题、回答、总结、判断默认挂在当前步骤下
- 用户临时追问默认挂在当前问题或当前步骤下

### 复合问题自动拆分

- 用户输入复合问题后自动拆分
- 原句保留为父 `question`
- 子问题显式可见且可编辑
- 排序优先按理解依赖，低置信度时退回原句顺序
- 支持撤销拆分或重排结果

### 树编辑

- `提升一级 / 降低一级` 可逆
- 节点提升到根节点直接子级时自动转为 `module`
- 原有子树、标签、引用、状态不丢失

### 资料引用

- 资料导入到全局资料区
- 引用时按需生成 `resource-fragment`
- 模块、问题、总结引用资料时不复制原文

### 搜索与导出

- 搜索默认只搜当前模块
- 当前模块无结果时明确提示，不静默扩大到全主题
- 用户可主动切换到全主题或资料区搜索
- 导出默认隐藏 `plan-step` 标题，但保留其子内容顺序并并入模块正文
- 用户可显式选择包含完整学习路径

## 手动验收重点

- 文本区是否始终是学习主舞台，而不是被结构工具反客为主
- 自动结构化结果是否显式可见，且容易修正
- 结构调整是否足够快，能否替代大量复制粘贴
- 搜索作用域是否清晰，不会让用户在错误范围里迷失
- 导出内容是否更像整理后的笔记，而不是原始系统日志

## 浏览器级 smoke

### 安装

- `pnpm install`
- `pnpm exec playwright install chromium`

### 运行

- `pnpm run test:e2e`

### 当前覆盖

- 应用可正常打开，关键主区域存在，不直接落入白屏 / 不可用状态
- 首次进入无历史数据时，可初始化最小工作区并进入可编辑状态
- 对工作区做最小可见修改后，刷新仍能恢复到已保存工作区
- 预置多模块工作区后，模块切换会带来文本主视图 / 结构视图的明显反馈
- 资料区、搜索 / 定位区、导出区面板可渲染；首版只验证结构存在与最小内容展示

### 刻意未覆盖

- 真实 AI 调用与外部模型服务
- 仍在变化中的文本连续输入、标签入口、资料创建入口等交互
- 对搜索结果文案、复杂编辑行为和资源创建流程的重断言

## 计划中的命令基线

工程初始化后，应至少统一以下命令：

- `pnpm run test`
- `pnpm run test:e2e`
- `pnpm run test:watch`
- `pnpm run coverage`
- `pnpm run typecheck`
- `pnpm run build`
